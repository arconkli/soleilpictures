# Capacitor source assets

These five files are the source-of-truth for the iOS / Android app icon
and splash screen. `@capacitor/assets` fans them out into every
platform-specific size (Android adaptive icon variants, iOS @2x/@3x,
splash for every screen ratio) when you run `npm run cap:assets`.

| File | What | Source |
|---|---|---|
| `icon-only.png` | Square app icon (used as foreground when adaptive icons aren't supported) | Copy of `boards/public/favicon.png` — the Soleil sun mark |
| `icon-foreground.png` | Android adaptive icon foreground layer | Same as above |
| `icon-background.png` | Android adaptive icon background layer (solid soleil-gold #ffa500) | Generated 1024×1024 |
| `splash.png` | iOS + Android splash, light/default | Sun mark centered on `#0a0a0c` (dark theme bg-0), 2732×2732 |
| `splash-dark.png` | Android dark-mode splash | Same as above (the brand is dark-themed by default) |

## Regenerating

If you want to swap the icon or splash artwork:

```bash
# Replace the .png files in this directory with your new artwork, then:
cd boards
npm run cap:assets  # only works after ios/ and android/ dirs exist
```

`@capacitor/assets` requires the native projects (`ios/` and `android/`)
to already exist — those are generated by `npm run cap:add:ios` and
`npm run cap:add:android`. Until then, these source files just sit
here ready to be consumed.

## Background colors

The npm script `cap:assets` is wired with:
- iconBackgroundColor = `#0a0a0c` (dark, matches app bg-0)
- splashBackgroundColor = `#0a0a0c`

Override the defaults at run time:
```bash
npx capacitor-assets generate --iconBackgroundColor '#ffa500'
```
